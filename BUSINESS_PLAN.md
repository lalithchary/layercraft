## B. LOGO DIRECTION

### Concept 1: Geometric Forge Mark
- **Description:** Abstract hexagonal shape with layered/stacked lines inside suggesting 3D printing layers. Minimal, one-color.
- **Style:** Geometric, technical, scalable
- **Colors:** Electric Blue (#00D4FF) on Dark (#0A0A0F)
- **SVG Direction:** Hexagon outline with 3 horizontal parallel lines inside, slightly offset to create depth illusion. Clean stroke width of 2-3px at standard size.

### Concept 2: Dimensional "P" Lettermark
- **Description:** The letter "P" constructed from isometric/3D perspective lines, giving it dimensional depth. Works as favicon and full logo.
- **Style:** Typographic, dimensional, futuristic
- **Colors:** Cyan (#00F0FF) + White (#FFFFFF) on Dark background
- **SVG Direction:** Letter P built with parallel lines at 30° angles to create isometric 3D effect. Negative space forms the letter.

### Concept 3: Layer Stack Icon (RECOMMENDED)
- **Description:** Three stacked rounded rectangles with slight perspective shift, suggesting layers being built up. Simple, instantly recognizable at any size.
- **Style:** Minimal, clean, modern
- **Colors:** Primary Cyan (#00E5FF), Accent (#7C3AED purple)
- **SVG Direction:** Three rounded rectangles (8px radius), each 40x8 units, stacked vertically with 4-unit gap. Each layer slightly offset to the right by 2 units, creating a subtle 3D stacking effect. Bottom layer full opacity, middle 80%, top 60%.

## E. 3D PREVIEW RECOMMENDATION

### Options Compared

| Option | Pros | Cons | Cost | GitHub Pages Compatible |
|--------|------|------|------|------------------------|
| **Three.js + STLLoader** | Full control, free, no dependency | Requires JS knowledge, STL only by default | Free | ✅ YES |
| **model-viewer (Google)** | Easy embed, AR support, glTF/GLB | Requires glTF format (not STL directly) | Free | ✅ YES |
| **Online3DViewer embed** | Supports STL/OBJ/3MF, open source | Heavier bundle, less customizable | Free | ✅ YES |
| **Sketchfab embed** | Beautiful renders, hosted | Requires upload to Sketchfab first, not for customer uploads | Free tier available | ✅ (embed only) |
| **ViewSTL.com embed** | Simple, free | External dependency, less control | Free | ✅ YES |

### RECOMMENDED APPROACH: Three.js with STL/OBJ Loader

**Why:** 
- Fully runs client-side (perfect for GitHub Pages)
- Customer uploads file → renders in browser immediately
- No server needed
- Supports STL (most common 3D print format)
- Can add OBJ support easily
- Full control over appearance

**Implementation:**
```
Customer uploads STL file → 
Browser reads file locally (FileReader API) → 
Three.js renders 3D preview in canvas → 
Customer can rotate/zoom → 
File dimensions displayed → 
"Get Quote" button with file details sent via form/WhatsApp
```

**Limitations:**
- File stays on client side only (no server storage on GitHub Pages)
- Large files (>50MB) may be slow on mobile
- 3MF support requires additional parser (can add later)
- No automatic quoting (quote is manual via WhatsApp/email)
- Cannot process/slice files server-side

**Fallback Approach:**
If Three.js is too complex to implement, use:
1. Simple file upload form (customer sends file)
2. Show file name + size + "We'll review and send preview"
3. Link to free online viewers: "Preview your file at viewstl.com"

### What CANNOT Be Done on GitHub Pages:
- ❌ Server-side file processing
- ❌ Automatic price calculation from file analysis
- ❌ File storage for later retrieval
- ❌ User accounts / order tracking
- ❌ Payment gateway integration (needs backend)

### Workarounds:
- Price calculation → Manual via WhatsApp after file review
- File storage → Customer sends file via WhatsApp/email/Google Form with file upload
- Payments → UPI QR code / bank transfer / Razorpay payment link (no website integration needed)

### FAQ Content

**Q: What file formats do you accept?**
A: We accept STL, OBJ, 3MF, and STEP files. STL is most common and recommended. If you have a different format, send it over and we'll check compatibility.

**Q: How long does printing take?**
A: Most orders are completed in 1-3 business days. Simple prints under 6 hours may ship same-day. We'll give you an estimated time with your quote.

**Q: What's the minimum order?**
A: For students, there's no minimum — we print single small parts starting at ₹49. For commercial orders, minimum is ₹99.

**Q: Can I visit the workshop?**
A: Yes! Walk-ins are welcome during business hours (Mon-Sat, 10AM-7PM). You can see your print being made.

**Q: How do I pay?**
A: We accept UPI (GPay, PhonePe, Paytm), bank transfer (NEFT/IMPS), and cash for walk-in orders.

**Q: Do you ship all over India?**
A: Yes. We ship pan-India via Speed Post and private couriers. Shipping starts at ₹80 depending on weight and location.

**Q: What if my print has a defect?**
A: We do quality checks on every print. If there's a defect on our end, we reprint for free. Share photos within 24 hours of delivery.

**Q: Can you help design my model?**
A: Absolutely. Send us a sketch, photo, or description and our team will create a print-ready 3D model. Design services start at ₹499.

**Q: What materials are strongest?**
A: For strength: ABS or PETG (FDM), Tough Resin (SLA). For flexibility: TPU. For general use: PLA is great and most affordable.

**Q: Is my file kept private?**
A: Yes. All files are treated as confidential. We don't share, display, or reuse any customer files without explicit permission.

### Icon Style
- Use Lucide Icons (free, consistent, SVG)
- Stroke width: 1.5px
- Color: Primary cyan or text secondary
- Size: 24px standard, 20px in cards

### Spacing System
- Base unit: 4px
- Small: 8px, 12px, 16px
- Medium: 24px, 32px, 40px
- Large: 48px, 64px, 80px
- Section gap: 80px desktop, 48px mobile

### Animations (Light, Performance-First)
- Page load: Fade-in-up (opacity 0→1, translateY 20px→0, 0.4s ease)
- Scroll: IntersectionObserver-triggered fade-in for sections
- Hover: Subtle scale (1.02) on cards, glow on buttons
- 3D viewer: Smooth orbit controls
- **No heavy libraries** — use CSS transitions + minimal JS

### Dark/Light Mode
- **Default: Dark mode** (matches futuristic aesthetic)
- Optional light mode toggle (swap background/text values)
- Use CSS custom properties for easy theming
